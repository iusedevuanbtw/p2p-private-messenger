# P2P Messenger VPN

Кроссплатформенный P2P мессенджер со встроенным VPN.

## Описание

Децентрализованный инструмент для общения с интегрированной виртуальной частной сетью. Позволяет создавать комнаты и подключаться к ним через прямое TCP-соединение с парольной защитой. Встроенный VPN на базе TUN/TAP обеспечивает виртуальную локальную сеть.

## Возможности

- P2P обмен сообщениями по TCP
- VPN через TUN/TAP интерфейсы
- Обнаружение комнат через UDP мультикаст
- SHA-256 аутентификация
- Приватные сообщения
- Отправка файлов
- Индикатор печати
- Уведомления на рабочий стол
- Замер задержки (пинг)
- История чата
- Подсветка упоминаний
- Статистика
- Цветной интерфейс
- Передача хоста комнаты
- Авто-скрытие оффлайн комнат

## Команды

| Команда | Описание |
|---------|----------|
| `/quit` | Выйти из чата |
| `/nick имя` | Сменить никнейм |
| `/clear` | Очистить экран |
| `/dice` | Бросить кубик |
| `/ping` | Замерить задержку |
| `/stats` | Статистика |
| `/online` | Список участников |
| `/msgto ник текст` | Приватное сообщение |
| `/hostto ник` | Передать хост комнаты |
| `/sendfile путь` | Отправить файл |

## Требования

- Python 3.8+
- Linux: модуль TUN/TAP, команда ip
- Права root для VPN

## Установка

    git clone https://github.com/iusedevuanbtw/p2p-private-messenger
    
    
    cd p2p-private-messenger
    
    
    chmod +x setup.sh
    
    sudo ./setup.sh

Запуск

    sudo p2pmsngr

Или:

    cd /opt/p2p_messenger_vpn
    source /opt/p2pvpn_env/bin/activate
    sudo python3 main.py

Архитектура

main.py — точка входа, управление сервисами
cli_menu.py — пользовательский интерфейс
 messenger.py — TCP соединения и аутентификация
discovery.py — поиск комнат через мультикаст
vpn.py — TUN/TAP интерфейс
auth.py — авторизация через центральный сервер
setup.sh — установщик
run.sh — лаунчер

Протокол

Поиск комнат: UDP мультикаст 224.0.0.187:48879, хосты анонсируют комнаты каждые 2 секунды.

Подключение: клиент выбирает комнату, TCP соединение, отправка JSON с названием комнаты и SHA-256 хешем пароля.

Типы сообщений:

 1 — текст

 2 — файл
3 — VPN пакет

5 — приватное сообщение

VPN: каждый участник создаёт TUN интерфейс с адресом 10.x.x.1/8, пакеты передаются внутри TCP.
Авторизация

Регистрация и вход через центральный сервер. Серверная часть не входит в открытый код и работает отдельно.
